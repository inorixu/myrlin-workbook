       - Class or module `OfflineQueue`:
         - `enqueueAction(action: Omit<QueuedAction, 'id' | 'timestamp' | 'retries'>): void` - adds to queue, persists to MMKV
         - `replayQueue(apiClient: MyrlinAPIClient): Promise<{ succeeded: number, failed: number }>` - iterates queue, replays each via the API client's raw fetch, removes on success, increments retries on failure, drops after 3 retries
         - `getQueueLength(): number`
         - `clearQueue(): void`
       - Persist queue to MMKV (key: `offline_queue`) so it survives app restart
       - Import MMKV from the existing store pattern (check how theme-store or server-store creates MMKV instances)

       - Show toast on successful replay: "{N} queued actions synced"
       - Update server-store connectionStatus to 'disconnected' when NetInfo reports no connectivity (only if currently 'connected', so it does not fight the health polling)
