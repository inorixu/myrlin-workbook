<interfaces>
<!-- Server API endpoints for workspace docs -->
```
GET    /api/workspaces/:id/docs              -> { docs: WorkspaceDocs }
PUT    /api/workspaces/:id/docs              -> { docs: WorkspaceDocs }  (full replacement)
POST   /api/workspaces/:id/docs/:section     -> { docs: WorkspaceDocs }  (add item to section)
PUT    /api/workspaces/:id/docs/:section/:idx -> { docs: WorkspaceDocs }  (update item at index)
DELETE /api/workspaces/:id/docs/:section/:idx -> { docs: WorkspaceDocs }  (delete item at index)

Sections: notes, goals, tasks, roadmap, rules
Each section is an array of items. Item shape varies by section:
- notes: { text: string }
- goals: { text: string, done: boolean }
- tasks: { text: string, done: boolean }
- roadmap: { text: string, status: 'planned' | 'active' | 'done' }
- rules: { text: string }
```

<!-- Server API endpoints for features -->
```
GET    /api/workspaces/:id/features              -> { features: Feature[] }
POST   /api/workspaces/:id/features              -> { feature: Feature }
PUT    /api/features/:id                         -> { feature: Feature }
DELETE /api/features/:id                         -> { success: boolean }
POST   /api/features/:id/sessions/:sessionId     -> { feature: Feature }
DELETE /api/features/:id/sessions/:sessionId      -> { feature: Feature }

Feature shape:
{
  id: string;
  workspaceId: string;
  name: string;
  description?: string;
  status: 'backlog' | 'active' | 'done';
  priority?: 'low' | 'medium' | 'high';
  sessionIds: string[];
  createdAt: string;
}
```

1. In `mobile/types/api.ts`, add these types (do NOT remove existing types):

```typescript
// Doc item types per section
interface DocNoteItem { text: string; }
interface DocGoalItem { text: string; done: boolean; }
interface DocTaskItem { text: string; done: boolean; }
interface DocRoadmapItem { text: string; status: 'planned' | 'active' | 'done'; }
interface DocRuleItem { text: string; }

type DocSection = 'notes' | 'goals' | 'tasks' | 'roadmap' | 'rules';

interface WorkspaceDocs {
  notes: DocNoteItem[];
  goals: DocGoalItem[];
  tasks: DocTaskItem[];
  roadmap: DocRoadmapItem[];
  rules: DocRuleItem[];
}

interface Feature {
  id: string;
  workspaceId: string;
  name: string;
  description?: string;
  status: 'backlog' | 'active' | 'done';
  priority?: 'low' | 'medium' | 'high';
  sessionIds: string[];
  createdAt: string;
}
```

Export all of these.

2. In `mobile/services/api-client.ts`, add these methods (import new types):
- `getWorkspaceDocs(workspaceId: string): Promise<{ docs: WorkspaceDocs }>` - GET /api/workspaces/:id/docs
- `addDocItem(workspaceId: string, section: DocSection, item: any): Promise<{ docs: WorkspaceDocs }>` - POST /api/workspaces/:id/docs/:section with body { text, done?, status? }
- `updateDocItem(workspaceId: string, section: DocSection, index: number, item: any): Promise<{ docs: WorkspaceDocs }>` - PUT /api/workspaces/:id/docs/:section/:index
- `deleteDocItem(workspaceId: string, section: DocSection, index: number): Promise<{ docs: WorkspaceDocs }>` - DELETE /api/workspaces/:id/docs/:section/:index
- `getFeatures(workspaceId: string): Promise<{ features: Feature[] }>` - GET /api/workspaces/:id/features
- `createFeature(workspaceId: string, data: { name: string; description?: string; status?: string; priority?: string }): Promise<{ feature: Feature }>` - POST /api/workspaces/:id/features
- `updateFeature(id: string, data: Partial<Feature>): Promise<{ feature: Feature }>` - PUT /api/features/:id
- `deleteFeature(id: string): Promise<{ success: boolean }>` - DELETE /api/features/:id

3. Create `mobile/hooks/useDocs.ts`:
- `useWorkspaceDocs(workspaceId: string)` - useQuery with queryKey ['workspace-docs', workspaceId], staleTime 10000
- `useAddDocItem()` - useMutation that invalidates ['workspace-docs'] on success
- `useUpdateDocItem()` - useMutation that invalidates ['workspace-docs'] on success
- `useDeleteDocItem()` - useMutation that invalidates ['workspace-docs'] on success
- `useFeatures(workspaceId: string)` - useQuery with queryKey ['features', workspaceId], staleTime 10000
- `useCreateFeature()` - useMutation that invalidates ['features'] on success
- `useUpdateFeature()` - useMutation that invalidates ['features'] on success
- `useDeleteFeature()` - useMutation that invalidates ['features'] on success
Follow exact same patterns as useSessions.ts (useAPIClient, useQueryClient).

4. Create `mobile/app/(tabs)/docs/_layout.tsx` with a basic Stack navigator (same pattern as other tab _layout files in the project, e.g. `more/_layout.tsx`). This enables future nested routes.
