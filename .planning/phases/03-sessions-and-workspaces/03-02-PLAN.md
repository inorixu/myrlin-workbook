**1. Create mobile/components/sessions/SessionActions.tsx:**

A wrapper component that presents an ActionSheet with session-specific actions. Props: `{ session: Session; visible: boolean; onClose: () => void; onNavigateToWorkspaces?: () => void }`.

Actions in the sheet (conditionally shown based on session.status):
- Start (if stopped/error/idle) - uses useStartSession mutation
- Stop (if running) - uses useStopSession mutation
- Restart (if running) - uses useRestartSession mutation
- Rename - shows a prompt modal (use ModalSheet with Input), calls useUpdateSession with { name }
- Move to Workspace - shows a picker (ModalSheet with workspace list from useWorkspaces), calls useUpdateSession with { workspaceId }
- Auto-title (AI) - calls api client autoTitle(), on success invalidates session query
- Summarize (AI) - calls api client summarize(), shows result in an alert/toast
- Save as Template - calls api client createTemplate() with session data mapped to template shape
- Delete - shows confirm dialog (use ActionSheet with destructive style), calls useDeleteSession, navigates back

Each action should show a Toast on success/error. Use useAPIClient() for direct API calls (autoTitle, summarize, createTemplate). Use mutation hooks for standard CRUD. All colors from useTheme().

**2. Create mobile/app/(tabs)/sessions/[id].tsx:**

Session detail screen. Gets session ID from route params via `useLocalSearchParams<{ id: string }>()`.

Layout (ScrollView):
- Header: Back button + session name (editable on tap) + overflow menu (three dots -> SessionActions)
- Status section: Large StatusDot + status label + start/stop/restart buttons (Button components, conditionally shown)
- Metadata section (SectionHeader + MetaRow list):
  - Workspace (name + color dot)
  - Working directory (monospace)
  - Command (monospace)
  - Created at (formatted date)
  - Last active (relative time, e.g. "3 minutes ago")
  - PID (if running)
- Tags section: Horizontal list of Badge components + "Add Tag" button. Tap add shows Input modal, submit calls useUpdateSession with updated tags array. Tap existing tag shows remove confirmation.
- Cost section (SectionHeader + cost summary):
  - Use useQuery with key ['session-cost', id] calling client.getSessionCost(id)
  - Show total cost formatted as "$X.XX"
  - TokenBar component showing input/output/cache breakdown
  - Model breakdown list
- Logs section (SectionHeader + recent logs):
  - Show last 20 logs from session.logs array
  - Each log: timestamp (monospace, muted) + message
  - Collapsible (show 5, "Show more" expands)
- Subagents section (if any):
  - Use useQuery with key ['session-subagents', id] calling client.getSessionSubagents(id)
  - Tree view: parent session -> child subagents with status dots
  - Collapsible section

Loading state: Skeleton placeholders for each section. Error state: EmptyState with retry. Use useTheme() for all colors.

**1. Create mobile/components/sessions/NewSessionModal.tsx:**

A ModalSheet-based form for creating a new session. Props: `{ visible: boolean; onClose: () => void; defaultWorkspaceId?: string }`.

Form fields:
- Name (Input, required)
- Workspace picker (horizontal Chip list from useWorkspaces, pre-select defaultWorkspaceId or first workspace)
- Working directory (Input with browse button - browse button shows directory picker using client.browse())
- Command (Input, defaults to "claude")
- Model (Input, optional, e.g. "opus", "sonnet")
- Template chips: horizontal ScrollView of template names from useQuery(['templates'], client.getTemplates). Tapping a template pre-fills command, workingDir, model from template data.
- Initial prompt (Input, multiline, optional)

Submit button calls useCreateSession mutation with CreateSessionInput. On success: close modal, show Toast. On error: show error Toast. All colors from useTheme().

**2. Create mobile/app/(tabs)/more/templates.tsx:**

Template list screen accessible from More tab menu.
- Header: "Templates" title
- FlashList of templates from useQuery(['templates'], client.getTemplates)
- Each template row: Card with template name (bold), command (monospace, muted), model, working dir
- Swipe-to-delete (or long-press -> ActionSheet with Delete option). Calls client.deleteTemplate(id), invalidates ['templates'].
- Empty state: EmptyState with "No templates" message, hint to save a session as template
- Pull-to-refresh

**3. Create mobile/app/(tabs)/more/session-manager.tsx:**

Bulk session management screen. SESS-18.
- Header: "Session Manager" title
- Summary cards: "X running", "X stopped", "X idle", "X error" (Badge components)
- "Stop All Running" button (destructive). On press: confirm dialog, then Promise.allSettled calling client.stopSession(id) for each running session. Show progress: "Stopping X/Y..." Toast. On complete: invalidate ['sessions'], show result Toast with success/failure counts.
- Session list: FlashList of all sessions with checkboxes for multi-select (future enhancement, for now just the bulk stop). Each row: SessionCard (compact variant or just name + status).

**4. Create mobile/app/(tabs)/more/recent.tsx:**

Recently active sessions screen. SESS-14.
- Header: "Recent Sessions" title
- FlashList rendering SessionCard items
- Data: useSessions('recent', { count: 20 })
- Pull-to-refresh
- Tap navigates to session detail
- Empty state: "No recent sessions"

All screens use useTheme() for colors. All have loading/error/empty states.
