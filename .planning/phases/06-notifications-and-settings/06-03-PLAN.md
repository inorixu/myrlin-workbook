**Settings store (mobile/stores/settings-store.ts):**
Create a Zustand store with MMKV persistence (same pattern as theme-store.ts):

```typescript
interface SettingsStoreState {
  notificationLevel: 'all' | 'errors' | 'none';
  hapticFeedback: boolean;
  confirmBeforeClose: boolean;
}

interface SettingsStoreActions {
  setNotificationLevel: (level: 'all' | 'errors' | 'none') => void;
  setHapticFeedback: (enabled: boolean) => void;
  setConfirmBeforeClose: (enabled: boolean) => void;
}
```

Persist all fields to MMKV with key `myrlin-settings`. Default: notificationLevel 'all', hapticFeedback true, confirmBeforeClose true.

**Settings screen (mobile/app/(tabs)/more/settings.tsx):**

Build a ScrollView-based settings screen with these sections, each using SectionHeader for the title:

**1. Appearance section:**
- "Theme" row showing current theme name. Tapping opens an inline theme picker grid (not a modal).
- Theme picker: 13 theme swatches in a 4-column grid. Each swatch is a small rounded square filled with that theme's `accent` color, with the theme name below. Active theme has a checkmark overlay and a border ring in accent color.
- Tapping a swatch calls `setTheme(themeId)`. The screen immediately re-renders with the new theme (live preview).

**2. Notifications section:**
- "Push Notifications" segmented control with three options: "All", "Errors Only", "None". Uses SegmentedControl component from UI library.
- Value reads from `useSettingsStore(s => s.notificationLevel)`, onChange calls `setNotificationLevel`.
- Brief description text below: "Controls which events trigger push notifications"

**3. General section:**
- "Haptic Feedback" toggle row (label + Toggle component). Reads/writes hapticFeedback from settings store.
- "Confirm Before Close" toggle row. Reads/writes confirmBeforeClose.

**4. Servers section:**
- "Manage Servers" row with chevron-forward icon. Tapping navigates to `/more/servers`.
- Show count of connected servers as a Badge.

**5. About section:**
- "Version" row showing `Application.nativeApplicationVersion` from `expo-application`. Non-interactive.
- "Check for Updates" button row. Tapping calls `Updates.checkForUpdateAsync()` from `expo-updates`. Shows result in a toast: "App is up to date" (success) or "Update available, restart to apply" (info) after calling `Updates.fetchUpdateAsync()` then `Updates.reloadAsync()`. Wrap in try/catch; in development builds where Updates is not available, show "Updates not available in dev builds" (info toast).
- "Build" row showing `Application.nativeBuildVersion` (non-interactive, small muted text).

Style every row consistently: 48px height, horizontal padding 16px, borderBottom 1px in borderSubtle color. Section gaps of 24px. Use theme.colors for all colors. Use fonts.sans.medium for labels, fonts.sans.regular for descriptions.

**Add renameServer to server-store.ts** if not present:
```typescript
renameServer: (id: string, name: string) => {
  set((state) => ({
    servers: state.servers.map((s) => s.id === id ? { ...s, name } : s),
  }));
}
```
