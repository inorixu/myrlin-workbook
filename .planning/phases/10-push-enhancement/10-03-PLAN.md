Create test/push-enhancement.test.js using Node's built-in test runner (`node:test` and `node:assert`).

**Test setup pattern:**
- Mock `global.fetch` before each test to intercept Expo Push API calls
- Track calls to fetch in an array for assertion
- Create a mock store object with:
  - `getPairedDevices()` returning test devices with pushTokens and pushPreferences
  - `updatePairedDevice(deviceId, updates)` tracking calls for assertion
  - `findDeviceByToken(token)` returning the matching test device
  - `state.sessions` with some running sessions (for badge count)
  - EventEmitter `on`/`emit` methods (use a real EventEmitter or simple mock)
- After each test, restore original fetch

**Test groups:**

1. **Retry tests:**
   - "retries on transient failure and succeeds": Mock fetch to throw on attempt 0, return ok on attempt 1. Assert sendPush resolves. Assert fetch called twice.
   - "gives up after max retries": Mock fetch to always throw. Assert sendPush resolves (does not throw). Assert fetch called 3 times.
   - "clears stale pushToken on DeviceNotRegistered": Mock fetch to return `{ data: [{ status: 'error', details: { error: 'DeviceNotRegistered' } }] }`. Assert updatePairedDevice was called with `{ pushToken: null }`.

   NOTE: Retry sleep delays would make tests slow. Override the sleep timing by either: (a) using very short delays in test (if the module exposes a config), or (b) mocking setTimeout to fire immediately. Prefer approach (b): save original setTimeout, replace with `(fn) => fn()`, restore after test.

2. **Batching tests:**
   - "single event sends full notification": Call queuePush once, then flushPushQueue. Assert fetch called with original title/body.
   - "multiple events send summary": Call queuePush 3 times with different notifications, then flushPushQueue. Assert fetch called once per device with summary title containing "3 updates".

3. **Preference filtering tests:**
   - "skips device with disabled preference": Create device with `pushPreferences: { sessionComplete: false }`. Call queuePush with `type: 'session:complete'`. Call flushPushQueue. Assert fetch NOT called (or called 0 times for that token).
   - "sends to device with enabled preference": Create device with `pushPreferences: { sessionComplete: true }`. Call queuePush with `type: 'session:complete'`. Call flushPushQueue. Assert fetch called for that device.
   - "defaults to true for unknown event type": Create device with minimal pushPreferences. Call queuePush with `type: 'custom:event'`. Call flushPushQueue. Assert fetch called.

**Important:** Use `require` with cache clearing to get a fresh push module per test group (or per test if needed). The module has internal state (pushQueue, flushTimer) that must be reset between test groups.

Clear the require cache for push.js before each test group:
