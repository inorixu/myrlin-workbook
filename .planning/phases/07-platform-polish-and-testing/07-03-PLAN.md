  <action>
    1. Install test dependencies if not already present:
       ```
       cd mobile
       npm install --save-dev jest @types/jest ts-jest
       ```
       If jest is already configured (check package.json for "jest" config), skip installation.

    2. Create `mobile/test/api-client.test.ts` with integration tests for the API client:

       Use a mock server approach (not a real server). Create a minimal mock using `jest.fn()` or a simple HTTP interceptor:

       ```typescript
       // Mock fetch globally for these tests
       const mockFetch = jest.fn();
       global.fetch = mockFetch;
       ```

       **Test suites:**

       a. **Session endpoints:**
          - `getSessions()` returns session array on 200
          - `getSessions()` throws on 401 (unauthorized)
          - `getSession(id)` returns single session on 200
          - `createSession(data)` sends POST with correct body
          - `deleteSession(id)` sends DELETE to correct URL
          - `startSession(id)` / `stopSession(id)` / `restartSession(id)` send correct methods
          - `renameSession(id, name)` sends PATCH with name in body
          - `moveSession(id, workspaceId)` sends PATCH with workspace

       b. **Workspace endpoints:**
          - `getWorkspaces()` returns workspace array
          - `createWorkspace(data)` sends POST
          - `deleteWorkspace(id)` sends DELETE

       c. **Cost endpoints:**
          - `getCosts(period)` sends correct query params for period filter

       d. **Task endpoints:**
          - `getTasks()` returns task array
          - Task CRUD operations send correct methods and bodies

       e. **Push endpoints:**
          - `registerPush(data)` sends POST with deviceToken and platform
          - `unregisterPush(token)` sends POST with token

       f. **Error handling:**
          - Network error (fetch throws TypeError) is caught and can be identified as network failure
          - 500 response throws server error
          - 404 response for unknown session throws not-found error

       g. **Auth header:**
          - All requests include `Authorization: Bearer {token}` header
          - Pairing request omits Authorization header

       Each test should:
       - Set up mockFetch.mockResolvedValueOnce with appropriate Response
       - Call the API client method
       - Assert the fetch was called with correct URL, method, headers, and body
       - Assert the return value matches expected shape

    3. Add test script to package.json if not present: `"test": "jest"`
       Add jest config if needed:
       ```json
       "jest": {
         "preset": "ts-jest",
         "testEnvironment": "node",
         "testMatch": ["**/test/**/*.test.ts"]
       }
       ```
