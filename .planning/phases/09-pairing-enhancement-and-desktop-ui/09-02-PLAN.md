**Step 1: Install qrcode**
Run `npm install qrcode` to add the qrcode library. This is a browser-compatible library that can generate SVG strings from data. It will be loaded via a script tag from node_modules or bundled. Since this is a vanilla JS SPA that serves from public/, we need the browser build. The library's browser build is at `node_modules/qrcode/build/qrcode.min.js`. Add a vendor route in server.js to serve it.

In server.js, add BEFORE the static middleware (search for `express.static`):
```javascript
// Vendor: serve qrcode browser bundle for client-side QR generation
app.get('/vendor/qrcode.min.js', (req, res) => {
  res.sendFile(path.join(__dirname, '..', '..', 'node_modules', 'qrcode', 'build', 'qrcode.min.js'));
});
```

In index.html, add a script tag before the app.js script tag:
```html
<script src="/vendor/qrcode.min.js"></script>
```

<task type="auto">
  <name>Task 2: Implement QR modal logic, auto-refresh, and device management in app.js</name>
  <files>src/web/public/app.js</files>
  <action>
Add the following methods to the CWMApp class. Place them after the existing settings-related methods (search for `settings-btn` event binding to find the right area).

**1. initPairMobile() method** - called from constructor or init:
- Bind click on `#pair-mobile-btn` to call `this.showPairMobileModal()`
- Bind click on `#pair-mobile-close-btn` to close the modal
- Bind click on `#pair-mobile-overlay` (backdrop) to close if clicking overlay itself
- Bind tab switching: click on `.pair-tab` buttons toggles `active` class and shows/hides tab content
- Call `this.updatePairBadge()` to show device count on the header button

**2. showPairMobileModal() method:**
- Show `#pair-mobile-overlay` (remove hidden attribute)
- Call `this.loadPairingCode()` to fetch and render QR
- Call `this.loadPairedDevices()` to populate devices tab
- Start auto-refresh timer: `this._pairRefreshTimer = setInterval(() => this.loadPairingCode(), 4 * 60 * 1000)` (every 4 minutes)
- Start countdown timer: update `#pair-timer` text every second showing "Refreshes in X:XX"

**3. closePairMobileModal() method:**
- Hide overlay (add hidden attribute)
- Clear `this._pairRefreshTimer` interval
- Clear countdown interval

**4. loadPairingCode() method:**
- `const res = await this.api('/api/auth/pairing-code')`
- Parse `res.qrPayload` (it's a JSON string)
- Generate QR SVG: `const svg = await QRCode.toString(res.qrPayload, { type: 'svg', width: 256, margin: 2, color: { dark: getComputedStyle(document.documentElement).getPropertyValue('--text').trim(), light: '#00000000' } })`
- Set `#pair-qr-container` innerHTML to the SVG string
- Parse the qrPayload JSON to extract urls
- Render detected URLs in `#pair-urls`:
  - For each URL in the urls object (lan, tailscale, tunnel, local), if non-null, show a row with the URL label and value
  - Each URL row has a small copy button that copies to clipboard
  - Use monospace font (var(--font-mono)) for URLs
- Store `res.expiresAt` for countdown display
- Reset countdown timer

**5. loadPairedDevices() method:**
- `const res = await this.api('/api/devices')`
- If `res.devices.length === 0`, show "No paired devices" message
- Otherwise, render each device as a card in `#pair-devices-list`:

```html
<div class="pair-device-card">
  <div class="device-icon">{platform emoji}</div>
  <div class="device-info">
    <div class="device-name">
      <span class="device-online-dot {online ? 'online' : ''}"></span>
      {deviceName}
    </div>
    <div class="device-meta">
      {platform} - Paired {relative time} - Last seen {relative time or 'Online'}
    </div>
    <div class="device-push-status">
      Push: {pushToken ? 'Registered' : 'Not registered'}
    </div>
  </div>
  <div class="device-actions">
    <button class="btn btn-ghost btn-sm" data-action="test-push" data-device-id="{deviceId}" {disabled if no pushToken}>Test Push</button>
    <button class="btn btn-ghost btn-sm btn-danger" data-action="revoke" data-device-id="{deviceId}">Revoke</button>
  </div>
</div>
```

- Use event delegation on `#pair-devices-list` for button clicks
- Platform emoji: iOS = Apple symbol or phone icon, Android = robot or phone icon. Use simple text: `platform === 'ios' ? '\uD83D\uDCF1' : '\uD83E\uDD16'` (mobile phone vs robot)

**6. handleDeviceAction(action, deviceId) method:**
- If action === 'test-push': POST to `/api/devices/${deviceId}/test-push`, show toast on success/failure
- If action === 'revoke': Show confirm dialog first ("Revoke {deviceName}? This will disconnect the device immediately."). On confirm: DELETE to `/api/devices/${deviceId}`, then reload device list and update badge

**7. updatePairBadge() method:**
- Fetch `/api/devices`, count devices
- If count > 0, show `#pair-badge` with count text
- If count === 0, hide badge

**8. Relative time helper** (if not already present):
- `timeAgo(isoString)` returns "just now", "5 minutes ago", "2 hours ago", "3 days ago", etc.

**9. Wire it up:**
- In the constructor or `init()` method, call `this.initPairMobile()` after auth is established
- Add keyboard shortcut: Escape closes the modal if open

**Important implementation notes:**
- Use `this.api(path, options)` for all fetch calls (it adds the Bearer token automatically)
- For the confirm dialog before revoke, use the existing confirm pattern from the codebase (check for a `showConfirm` or similar method, or use window.confirm as fallback)
- Toast notifications: use `this.showToast(message, type)` if it exists, otherwise `this.notify(message)` or similar existing pattern
- The QR SVG dark color should use the current theme's text color so it's visible in both light and dark themes. Use transparent background so the modal background shows through.
