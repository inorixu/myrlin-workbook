  <action>
    1. Create `mobile/app/_layout.tsx` (Root Layout):
       - Call `SplashScreen.preventAutoHideAsync()` at module scope (top of file, outside component). This is CRITICAL to prevent FOUT.
       - Use `useFonts` hook to load all 7 font variants:
         - PlusJakartaSans: Regular (400), Medium (500), SemiBold (600), Bold (700) from @expo-google-fonts/plus-jakarta-sans
         - JetBrainsMono: Regular, Medium, Bold from require('../assets/fonts/...')
       - Read theme from `useThemeStore((s) => s.theme)`
       - In useEffect: when fontsLoaded or fontError, call `SplashScreen.hideAsync()`
       - If !fontsLoaded and !fontError, return null (splash stays visible)
       - Render: `<ThemeProvider theme={theme}><Stack screenOptions={{ headerShown: false }}><Stack.Screen name="(tabs)" /></Stack></ThemeProvider>`
       - The ThemeProvider wrapping here ensures ALL descendant screens get theme context

    2. Create `mobile/app/(tabs)/_layout.tsx` (Tab Layout):
       - Use `useTheme()` hook to read current theme
       - Configure `<Tabs>` with 5 screens: sessions, tasks, costs, docs, more
       - Style tab bar: backgroundColor = theme.colors.mantle, borderTopColor = theme.colors.surface0
       - Active tint = theme.colors.accent, inactive tint = theme.colors.overlay1
       - Header style: backgroundColor = theme.colors.base, tintColor = theme.colors.text
       - Header title font: PlusJakartaSans-SemiBold
       - Tab icons: Use Expo's built-in icon library (@expo/vector-icons) with appropriate icons for each tab (terminal/code for Sessions, checklist for Tasks, chart for Costs, document for Docs, ellipsis/menu for More)

    3. Create 5 placeholder tab screens (sessions/index.tsx, tasks/index.tsx, costs/index.tsx, docs/index.tsx, more/index.tsx):
       - Each renders a SafeAreaView with themed background (theme.colors.base)
       - Center text showing the tab name using themed colors and PlusJakartaSans font
       - These are placeholders that will be replaced in Phase 3-6 but must render correctly for navigation testing

    4. Create `mobile/types/navigation.ts` with typed route parameter types for future screens. For now, define the tab routes and placeholder param types for session detail and task detail screens that will be added later.

    IMPORTANT: The `SplashScreen.preventAutoHideAsync()` call MUST be at module scope. If placed inside a component or useEffect, the splash screen will auto-hide before fonts load, causing FOUT. This is the most common mistake per the research pitfalls.

    Build all 17 UI primitive components. Every component MUST:
    - Read colors from `useTheme()` hook (NEVER hardcode hex values)
    - Use StyleSheet.create outside the render function (or memoize with useMemo if theme-dependent)
    - Have TypeScript props interface exported alongside the component
    - Have a JSDoc comment describing the component's purpose

    Component specifications:

    1. **Button** (primary, ghost, danger variants; sm, md sizes; loading, disabled, icon props)
       - Pressable with pressed opacity 0.85, disabled opacity 0.5
       - Primary: accent bg, crust text. Ghost: transparent bg, text color. Danger: red bg, crust text.
       - Loading state shows ActivityIndicator

    2. **Badge** (variant: default, success, warning, error, info; children text)
       - Small pill with colored bg and contrasting text
       - Used for session counts, status labels

    3. **StatusDot** (status: running, stopped, error, idle; size: sm, md)
       - Circular dot with color mapped to status (green=running, overlay1=stopped, red=error, yellow=idle)
       - Animated pulse for running status using Reanimated

    4. **Card** (children, onPress optional, elevated boolean)
       - Surface0 bg, borderSubtle border, radius.lg
       - Optional onPress makes it a Pressable with hover feedback
       - Elevated variant adds shadow.md

    5. **Input** (label, placeholder, value, onChangeText, error, secureTextEntry, multiline)
       - TextInput with surface0 bg, borderDefault border, text color
       - Error state: red border, error text below
       - Label text in textSecondary

    6. **Toggle** (value, onValueChange, label, description)
       - Switch with accent trackColor when on, surface1 when off
       - Label and optional description text

    7. **SegmentedControl** (segments: string[], selectedIndex, onSelect)
       - Horizontal row of segments with active indicator (accent bg)
       - Inactive segments: transparent bg, textSecondary

    8. **EmptyState** (icon: ReactNode, title, description, action?: { label, onPress })
       - Centered layout with icon, title in textPrimary, description in textSecondary
       - Optional action button at bottom

    9. **SectionHeader** (title, action?: { label, onPress })
       - Title in textSecondary, uppercase, fontSize xs, semibold
       - Optional right-aligned action link

    10. **Skeleton** (width, height, borderRadius)
        - Animated shimmer effect using Reanimated (opacity oscillation between 0.3 and 0.7)
        - Surface1 background color
        - Used as loading placeholder for cards, text, avatars

    11. **Toast** (message, variant: success, error, info, warning; visible, onDismiss)
        - Positioned at top, slides in from top with Reanimated
        - Auto-dismisses after 3 seconds
        - Colored left border matching variant

    12. **SearchBar** (value, onChangeText, placeholder, onSubmit)
        - TextInput with search icon (magnifying glass) prefix
        - Surface0 bg, radius.full for pill shape
        - Clear button when value is non-empty

    13. **MetaRow** (label, value, icon?: ReactNode)
        - Horizontal row: icon + label (textSecondary) on left, value (textPrimary) on right
        - Used in session detail, task detail for metadata display

    14. **TokenBar** (input, output, cacheRead, cacheWrite proportions)
        - Horizontal stacked bar showing token type proportions
        - Colors: blue for input, green for output, teal for cacheRead, peach for cacheWrite
        - Fixed height, radius.sm

    15. **ModalSheet** (visible, onClose, title, children)
        - Bottom sheet modal with handle bar at top
        - Mantle background, animated slide up with Reanimated
        - Backdrop overlay on press closes

    16. **Chip** (label, selected, onPress, color?)
        - Small pill for filter chips (workspace filter, tag filter)
        - Selected: accent bg, crust text. Unselected: surface0 bg, textSecondary

    17. **ActionSheet** (visible, onClose, actions: { label, icon?, destructive?, onPress }[])
        - iOS-style action sheet rendered as bottom modal
        - Destructive actions in red
        - Cancel button at bottom

    18. Create `mobile/components/ui/index.ts` barrel export re-exporting all 17 components.

    IMPORTANT: Styles that depend on theme props must NOT use StyleSheet.create inside the render. Either use a getStyles(theme) function called outside render, or use inline styles with useMemo. The design doc and research both warn about this anti-pattern.
