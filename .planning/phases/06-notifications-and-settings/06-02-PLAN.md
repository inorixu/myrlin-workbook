**API client (mobile/services/api-client.ts):**
Add two methods to MyrlinAPIClient:

```typescript
/** Register a push notification token with the server */
async registerPush(data: PushRegisterInput): Promise<{ success: boolean }> {
  return this._fetch('/api/push/register', {
    method: 'POST',
    body: JSON.stringify(data),
  });
}

/** Unregister a push notification token */
async unregisterPush(deviceToken: string): Promise<{ success: boolean }> {
  return this._fetch('/api/push/unregister', {
    method: 'POST',
    body: JSON.stringify({ deviceToken }),
  });
}
```

**Server store (mobile/stores/server-store.ts):**
Add `pushToken: string | null` to ServerStoreState (transient, not persisted). Add `setPushToken(token: string | null)` action. This tracks the current device's Expo push token so we can unregister on server switch.

**Push service (mobile/services/push-service.ts):**
Create a new file that handles push notification setup:

1. `requestPushPermission()`: Calls `Notifications.requestPermissionsAsync()`. Returns boolean.
2. `getExpoPushToken()`: Calls `Notifications.getExpoPushTokenAsync({ projectId: Constants.expoConfig?.extra?.eas?.projectId })`. Returns the token string or null on failure.
3. `configurePushChannels()`: On Android, create a notification channel "session-events" with importance HIGH.

Import from `expo-notifications` and `expo-constants`.

**Push hook (mobile/hooks/usePush.ts):**
Create `usePushNotifications()` hook that:

1. On mount (useEffect with empty deps):
   - Call `configurePushChannels()` (Android channel setup)
   - Set notification handler: `Notifications.setNotificationHandler({ handleNotification: async () => ({ shouldShowAlert: true, shouldPlaySound: true, shouldSetBadge: true }) })`

2. On active server change (useEffect watching activeServerId + connectionStatus):
   - If connected and has active server:
     - Request permission via `requestPushPermission()`
     - If granted, get token via `getExpoPushToken()`
     - Store token in server store via `setPushToken(token)`
     - Register with server via `client.registerPush({ deviceToken: token, platform: Platform.OS })`
   - If server changed (previous token exists), unregister old token from previous server (best-effort)

3. Notification response listener (tap handler):
   - Use `Notifications.addNotificationResponseReceivedListener`
   - Extract `data` from notification content as `PushNotificationData`
   - Navigate using `router.push()`:
     - type 'session' with sessionId: `router.push(`/sessions/${data.sessionId}`)`
     - type 'task' with taskId: `router.push(`/data/tasks`)`
     - type 'conflict': `router.push(`/more/conflicts`)`

4. Cleanup: remove listeners on unmount.

The hook takes no arguments; it reads the API client from useAPIClient() and router from useRouter().
