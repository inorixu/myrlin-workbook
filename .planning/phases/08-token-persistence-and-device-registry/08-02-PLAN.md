Create a new file `src/web/device-manager.js` with device management routes.

IMPORTANT: Follow the code style of existing modules (pairing.js, push.js). Add SPDX header, JSDoc on every function, file header comment.

```javascript
/**
 * Device management module for paired mobile devices.
 *
 * Provides CRUD endpoints for managing paired devices:
 *   GET    /api/devices              - List all paired devices
 *   GET    /api/devices/:deviceId    - Get single device details
 *   PUT    /api/devices/:deviceId    - Update device (name, push prefs)
 *   DELETE /api/devices/:deviceId    - Revoke device (invalidate token)
 *   POST   /api/devices/:deviceId/test-push - Send test notification
 *
 * SPDX-License-Identifier: AGPL-3.0-only
 */
```

Implement `setupDeviceRoutes(app, { requireAuth, getStore, removeToken, sendPush, getSSEClients })`:

**GET /api/devices** (requireAuth):
- Get all paired devices from store.getPairedDevices()
- For each device, add `isOnline` field by checking if any SSE client has a matching token
- Strip the `token` field from the response (security: don't expose tokens via API)
- Return `{ devices: [...] }`

**GET /api/devices/:deviceId** (requireAuth):
- Find device by deviceId
- 404 if not found
- Add isOnline, strip token
- Return device object

**PUT /api/devices/:deviceId** (requireAuth):
- Find device by deviceId, 404 if not found
- Only allow updating: `deviceName` (string, max 100 chars), `pushPreferences` (object with boolean values)
- Validate deviceName: max 100 chars, trim whitespace
- Validate pushPreferences: only known keys, boolean values
- Call store.updatePairedDevice(deviceId, updates)
- Return updated device (stripped of token)

**DELETE /api/devices/:deviceId** (requireAuth):
- Find device by deviceId, 404 if not found
- Remove token from activeTokens via removeToken(device.token)
- Close any active SSE connections for this token: iterate getSSEClients(), find any whose associated token matches, call res.end() and remove from set
- Remove device from store via store.removePairedDevice(deviceId)
- Return `{ success: true }`

**POST /api/devices/:deviceId/test-push** (requireAuth):
- Find device by deviceId, 404 if not found
- Check device has a pushToken, 400 if not
- Send a test push via sendPush with title "Test notification" and body "Push notifications are working for [deviceName]"
- For test push, send directly to this device only (build a single message to device.pushToken, use fetch to Expo Push API directly rather than sendPush which goes to all devices)
- Return `{ success: true }`

**Helper: stripToken(device)**
- Returns a shallow copy of the device object with the `token` field removed
- Used in all GET responses

**Helper: isDeviceOnline(device, sseClients)**
- Check if any SSE client's associated token matches device.token
- This requires SSE clients to track their auth token (see Task 2)
